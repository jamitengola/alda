# Good First Issues

This document lists beginner-friendly contribution ideas for ALDA.

These tasks are useful for contributors who want to understand the project while improving documentation, tests, prompts and developer experience.

## Documentation

- Improve the README with screenshots.
- Add a short demo flow explaining how to use mock mode.
- Add Portuguese documentation for local communities.
- Add a quick explanation of each AI provider mode.
- Add troubleshooting notes for common setup problems.

## Testing

- Add tests for the AI provider selection logic.
- Add tests for API routes.
- Add tests for prompt formatting helpers.
- Add basic UI tests for the main pages.

## Developer experience

- Add Docker support for local development.
- Improve `.env.example` comments.
- Add a setup script for first-time contributors.
- Add Makefile or npm scripts for common commands.

## Product improvements

- Add meeting templates for sales, interviews and negotiation.
- Add export options for follow-up drafts.
- Improve empty states in the UI.
- Improve accessibility labels and keyboard navigation.

## AI and prompts

- Add prompt examples for different meeting types.
- Add evaluation examples for generated summaries.
- Improve local model instructions for Ollama.
- Add examples for Portuguese-language meeting coaching.

## Suggested labels

Useful labels for GitHub Issues:

- `good first issue`
- `documentation`
- `bug`
- `enhancement`
- `help wanted`
- `ai`
- `desktop`
- `accessibility`
