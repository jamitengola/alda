# ALDA Roadmap

This roadmap organizes the next steps for ALDA as an open source AI meeting coach.

## Phase 1 — Foundation

- [x] Create the initial Next.js application.
- [x] Add Electron desktop mode.
- [x] Add AI provider abstraction.
- [x] Support mock AI mode.
- [x] Add Ollama support.
- [x] Add OpenAI-compatible provider support.
- [x] Add README, license, security notes and contribution guide.
- [x] Add GitHub Actions CI.

## Phase 2 — Product maturity

- [ ] Add authentication with email code.
- [ ] Improve automated tests.
- [ ] Add prompt evaluation examples.
- [ ] Add meeting templates by context: sales, interview, negotiation, leadership and support.
- [ ] Add export options for summaries and follow-ups.
- [ ] Improve accessibility and keyboard navigation.
- [ ] Add screenshots and demo media to the README.

## Phase 3 — Integrations

- [ ] Add calendar integration for automatic meeting preparation.
- [ ] Add email integration for follow-up drafts.
- [ ] Add CRM integration examples.
- [ ] Add webhook support for automation platforms.

## Phase 4 — Desktop experience

- [ ] Improve desktop overlay experience.
- [ ] Add global shortcut support.
- [ ] Add local meeting history.
- [ ] Package stable desktop releases.

## Phase 5 — Community and open source

- [ ] Add issue templates.
- [ ] Add pull request template.
- [ ] Add `good first issue` examples.
- [ ] Publish contributor guide examples.
- [ ] Create release notes for stable versions.
- [ ] Add documentation for Portuguese-speaking developer communities.

## Suggested first issues

- Add screenshots to the README.
- Add unit tests for the AI provider layer.
- Improve `.env.example` documentation.
- Add Docker support for local development.
- Add examples for meeting preparation prompts.
